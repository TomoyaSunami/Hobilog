# 要件定義 v1.0（MVP）

## 0. アプリ概要

個人利用向けの習慣トラッキングPWA。

筋トレ・読書・勉強など複数の習慣を登録し、日々の実施状況や記録量を記録・可視化する。

スマホ利用前提。
UIはライトSF・ソフトグラスをベースにしつつ、毎日使いやすい控えめな演出にする。
ダークテーマはMVP初期実装では必須にせず、将来のテーマ切り替え候補として扱う。

---

# 1. MVP方針

## 1.1 初期データ

MVPでは初回起動時に以下の3習慣をプリセットする。

| 習慣名 | アイコン | テーマカラー |
| ------ | -------- | ------------ |
| 筋トレ | Dumbbell | Blue |
| 読書 | BookMarked | Pink |
| 勉強 | BookOpen | Cyan |

アイコンは `lucide-react` の線画アイコンから選択する。
テーマカラーは固定パレットから選択する。

テーマカラーの固定パレットは以下とする。

* Blue
* Cyan
* Pink
* Purple
* Green
* Amber
* Red
* Orange
* Slate

# 2. 機能要件

## 2.1 習慣管理

### 習慣登録

登録項目：

* 習慣名
* アイコン
* テーマカラー
* 記録方法
* カスタム単位（記録方法がカスタムの場合のみ）

### 習慣編集

編集項目：

* 名前変更
* アイコン変更
* 色変更
* 記録方法変更
* カスタム単位変更

### 習慣削除

* 削除確認ダイアログを表示する
* 削除時は対象習慣に紐づく記録も削除する

---

## 2.2 実施記録

### 記録内容

1日につき習慣ごとに1レコード。
同一習慣・同一日付の記録は新規追加ではなく上書きする。

| 項目 | 内容 |
| ---- | ---- |
| 日付 | 実施日 |
| 習慣ID | 対象習慣 |
| 実施フラグ | 実施したか |
| 記録量 | 習慣の記録方法に応じた数値 |
| メモ | 任意 |
| 更新日時 | 最終更新日時 |

記録方法は習慣ごとに設定する。

| 記録方法 | 単位 | 入力 |
| -------- | ---- | ---- |
| 実施のみ | なし | 数値入力なし |
| 時間 | 分 | 0以上の整数 |
| 回数 | 回 | 0以上の整数 |
| 距離 | km | 0以上の小数第1位まで |
| ページ数 | ページ | 0以上の整数 |
| カスタム | 任意 | 0以上の小数第1位まで |

### 入力ルール

* 過去日と当日は記録可能
* 未来日は記録不可
* 記録方法が `実施のみ` の場合、数値入力欄は表示しない
* それ以外の記録方法では、記録方法に応じた数値入力欄を表示する
* 記録量が未入力の場合は `0` として保存する
* 記録量は `0` 以上とする
* メモは任意

### 操作

* ホーム画面の「記録」「編集」ボタンから記録入力画面を開く
* 初期値は `実施ON / 記録量0 / メモ空`
* 記録保存
* 記録編集
* 記録削除

---

## 2.3 ログ画面

目的：過去記録をカレンダー形式で確認・編集する。

### 表示

* 月間カレンダー
* 日付ごとの実施状況
* 選択日の全習慣一覧
* 実施済み・未実施の両方を表示

### 操作

* 日付タップで対象日の記録一覧を表示する
* 各習慣をタップして記録入力画面を開く
* 過去日の記録追加・編集・削除ができる
* 未来日は選択できても記録保存は不可、または非活性表示にする

---

## 2.4 可視化

### 連続実施日数

連続実施日数は習慣ごとに計算する。
MVPでは全体streakは計算しない。

表示：

* 習慣ごとの現在の連続日数
* 習慣ごとの最長連続日数

判定：

* `実施フラグ = true` の日を実施日とする
* 1日でも未実施の日があると連続日数は途切れる
* 週3目標などの柔軟な目標頻度はMVP対象外

---

### 累計グラフ

#### グラフ仕様

* 折れ線グラフ
* 横軸：日付
* 縦軸：記録方法が `実施のみ` の場合は累計実施日数、それ以外の場合は累計記録量

#### 表示形式

* 選択中の習慣のみ表示する
* 線色は習慣カラーに連動する
* 期間切り替えとして `1ヶ月` / `3ヶ月` / `6ヶ月` / `1年` / `全期間` を表示する
* `全期間` は年フィルターを無視し、選択中習慣の最古の実施記録日から今日までを表示する
* 記録方法が `実施のみ` 以外の場合、グラフ見出しと分析指標は記録量ベースの表示に切り替える

---

### 年間ヒートマップ

#### 表示形式

GitHub Contribution Graph風。

#### 色の意味

* 未実施：薄いグレー
* 実施：選択中習慣のテーマカラー

#### 切り替え

* 対象習慣

# 3. UI/UX要件

## デザインコンセプト

ライトSF・ソフトグラス。

ただし、MVPでは日常利用のしやすさを優先し、発光やアニメーションは控えめにする。

### UI特徴

* ライトテーマ
* ガラスUI
* 淡いネオンアクセント
* 軽い発光エフェクト
* 滑らかなアニメーション
* ミニマル構成
* スマホ片手操作を前提にした下部固定ナビゲーション

### イメージ方向

* GitHub Contribution Graph
* ライトSF HUD
* SAO系UI
* PSYCHO-PASS風ダッシュボード
* 毎日使える控えめな近未来UI

---

# 4. 技術要件

## アプリ形式

PWA。

### 対応

* スマホブラウザ
* ホーム画面追加
* インストール可能なPWA

### MVPで求めないこと

* 本格的なオフラインファースト対応
* バックグラウンド同期
* プッシュ通知

---

## データ保存

MVPでは `localStorage` に保存する。

### 保存対象

* 習慣一覧
* 実施記録一覧
* アプリ設定

### 保存キー案

* `hobilog:habits`
* `hobilog:records`
* `hobilog:settings`

---

# 5. MVP対象外

今回は実装しない：

* ログイン
* クラウド同期
* SNS共有
* 通知
* ストップウォッチ
* AI分析
* PC最適化
* 週単位・曜日単位の目標頻度設定
* 全体streak計算
* IndexedDB対応
