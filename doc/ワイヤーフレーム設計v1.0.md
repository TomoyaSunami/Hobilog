# ワイヤーフレーム設計 v1.0

## v1.0 変更点

* `分析画面サンプル画像.png` をもとに分析画面を詳細化
* 分析画面に以下を追加
  * 対象習慣プルダウン
  * 年フィルター
  * 年間ヒートマップ
  * 累計グラフの期間切り替え（全期間を含む）
  * 6項目の分析指標
* UI方向性をサンプル画像に合わせて、ライト基調のソフトグラスUIに寄せる
* v0.2のダークSF表現は将来のテーマ切り替え候補として扱う

---

## 画面構成

```text
1. ホーム画面
2. 記録入力画面
3. ログ画面
4. 分析画面
5. 習慣管理画面
```

## 画面名称・操作配置ルール

| 画面 | 下部タブ名 | コンテンツ上部に置く情報・操作 |
| ---- | ---------- | ------------------------------ |
| ホーム画面 | ホーム | 今日の進捗カード内に日付を表示 |
| ログ画面 | ログ | 月間カレンダーの月移動操作を表示 |
| 分析画面 | 分析 | 対象習慣カード内に年フィルターを表示 |
| 習慣管理画面 | 習慣 | 習慣追加ボタンを一覧上部に表示 |
| 記録入力画面 | なし | モーダル内に `記録入力` を表示 |

* 下部タブは短い日本語名で統一する
* メイン画面では、フッターと重複する画面名だけのヘッダーは表示しない
* 画面固有の操作や状態は、関連するカード内に配置する

# 1. ホーム画面

目的：今日やる習慣をすぐ確認し、記録入力へ進む。

```text
┌────────────────────────┐
│ 今日の進捗   2026.05.27 Wed │
│ 3 habits / 1 done      │
│ █████░░░░░             │
├────────────────────────┤
│ [Icon] 筋トレ           │
│ Streak 5 days          │
│ Today: Done / 30 min   │
│ [編集]                  │
├────────────────────────┤
│ [Icon] 読書             │
│ Streak 2 days          │
│ Today: Not yet         │
│ [記録]                  │
├────────────────────────┤
│ [Icon] 勉強             │
│ Streak 0 days          │
│ Today: Not yet         │
│ [記録]                  │
├────────────────────────┤
│ ホーム | ログ | 分析 | 習慣 │
└────────────────────────┘
```

## 表示項目

* 今日の日付
* 今日の達成数
* 今日の進捗バー
* 習慣カード一覧
* 各習慣のアイコン
* 各習慣の現在連続日数
* 当日の記録状態
* 記録ボタン

## 挙動

* 初期習慣は「筋トレ」「読書」「勉強」
* 未記録の場合は「記録」、当日記録済みの場合は「編集」ボタンを表示する
* 「記録」「編集」ボタンを押すと記録入力画面へ遷移する
* 記録入力画面の初期値は `実施ON / 記録量0 / メモ空`
* 当日記録済みの場合も同じ画面を開き、既存記録を編集する

# 2. 記録入力画面

目的：選択した習慣の実施内容を入力・編集する。

```text
┌────────────────────────┐
│ 記録入力                │
├────────────────────────┤
│ 習慣                    │
│ [Icon] 読書             │
├────────────────────────┤
│ 日付                    │
│ [ 2026/05/27 ]          │
├────────────────────────┤
│ 実施した                │
│ [ ON ]                  │
├────────────────────────┤
│ ページ数                │
│ [ 0 ] ページ            │
├────────────────────────┤
│ メモ                    │
│ [                      ]│
├────────────────────────┤
│ [保存する]              │
│ [削除]                  │
└────────────────────────┘
```

## 入力項目

* 実施日
* 実施有無
* 記録方法に応じた記録量
* メモ 任意

## 入力ルール

* 過去日と当日は保存可能
* 未来日は保存不可
* 記録方法が `実施のみ` の場合、数値入力欄は表示しない
* それ以外の記録方法では、記録方法に応じた数値入力欄を表示する
* 記録量が未入力の場合は `0` として保存
* 記録量は `0` 以上
* 同一習慣・同一日付の記録は上書き保存
* 削除ボタンは既存記録がある場合のみ表示

### 対象習慣

```text
筋トレ
読書
勉強
```

### 対象習慣プルダウン

* 選択中の習慣アイコン、習慣名を表示する
* 右端に開閉アイコンを表示する
* 展開時は白背景のグラス風メニューを表示する
* メニュー内の各項目には習慣アイコン、習慣名を表示する
* 選択中の項目にはチェックアイコンを表示する
* カード右上に独立したカラーバッジは表示しない

### 年

* 現在年を初期表示する
* 選択肢には、現在年と最古の記録年までの年を連続して表示する
  * 例：2023年、2025年、2026年に記録がある場合、2024年も表示する
* 未来年は表示しない
* 保存済みの選択年が選択肢に存在しない場合は、選択可能な最新年に補正する
* 選択年に記録がない場合は、折れ線グラフ部分に「この年のデータはありません」を表示する

* 選択中習慣の実施有無を表示する
* 選択中習慣のテーマカラーでセルを表示する
* 月ごとの独立ブロックには分けない
* 週を1行、曜日を7列として縦方向に並べる
* ヒートマップグリッドはカード内で中央寄せにする
* 年間グリッドはカード内で縦スクロール可能にし、スクロールバーを表示する
* スクロールバーは通常時ニュートラルグレー、hover時のみ控えめなブルーにする
* 実施フラグが `false` の記録は未実施として扱う
* 記録がない日も未実施として扱う

```text
┌────────────────────────────┐
│ 累計記録量の推移 (i) [1ヶ月][3ヶ月][6ヶ月][1年][全期間] │

│ 分析指標                    │
```

### 表示項目

* セクションタイトル
* インフォメーションアイコン
* 期間切り替え
  * 1ヶ月
  * 3ヶ月
  * 6ヶ月
  * 1年
  * 全期間
* 折れ線グラフ
* 凡例
* タップ時のツールチップ
* 分析指標

### グラフルール

* 横軸：日付
* 縦軸：記録方法が `実施のみ` の場合は累計実施日数、それ以外の場合は累計記録量
* 選択中習慣のみ表示する
* 線色は習慣カラーに連動する
* 記録方法が `実施のみ` の場合、見出しは `累計実施日数の推移` とする
* 記録方法が `実施のみ` 以外の場合、見出しは `累計記録量の推移` とし、縦軸・ツールチップの単位は記録方法の単位に連動する
* `1ヶ月` / `3ヶ月` / `6ヶ月` / `1年` は選択中の年と現在日を基準に表示範囲を絞る
* `全期間` は年フィルターを無視し、選択中習慣の最古の実施記録日から今日までを表示する
* 年を変更した場合、`1年` 表示では対象年の1月1日から12月31日までを表示する

### ツールチップ

```text
2025/05/26
● 筋トレ        48回
```

* グラフ上の点または近傍をタップすると表示する
* 選択中習慣のみ表示する

## 分析指標

累計グラフカード内のグラフ下に配置する。対象期間は同カード上部の期間切り替えと連動する。

```text
│ 分析指標                    │
│ [Flame] 現在連続      5日    │
│ [Crown] 最長連続     12日    │
│ [Clock] 総記録量     91回    │
│ [Calendar] 実施日数  78日    │
│ [Star] 平均/実施日  1.8回   │
│ [Chart] 実施率       78%    │
```

### 表示項目

* 現在連続
* 最長連続
* 総実施日数 または 総記録量
* 実施日数
* 平均/日 または 平均/実施日
* 実施率

### 集計ルール

* `現在連続`
  * 選択中習慣の現在連続日数
* `最長連続`
  * 選択中習慣の最長連続日数
* `総実施日数`
  * 記録方法が `実施のみ` の場合に表示する
  * 対象期間内の `実施フラグ = true` の記録数
  * 選択中習慣のみ
* `総記録量`
  * 記録方法が `実施のみ` 以外の場合に表示する
  * 対象期間内の `実施フラグ = true` の記録量合計
  * 単位は記録方法に連動する
* `実施日数`
  * 対象期間内で選択中習慣を実施したユニーク日数
* `平均/日`
  * 記録方法が `実施のみ` の場合に表示する
  * `総実施日数 ÷ 対象期間の日数`
  * 小数第1位まで表示
* `平均/実施日`
  * 記録方法が `実施のみ` 以外の場合に表示する
  * `総記録量 ÷ 実施日数`
  * 小数第1位まで表示
* `実施率`
  * `実施日数 ÷ 対象期間の日数 × 100`
  * 整数パーセントで表示

# 7. UI方向性

## 基本コンセプト

```text
ライトSF
ソフトグラス
淡いネオンアクセント
白背景ベース
読みやすいダッシュボード
```

## カード

```text
背景：白〜半透明白
枠線：薄いBlue Gray
角丸：16〜20px
影：淡いBlue系シャドウ
余白：広め
```

## 配色

```text
背景：#F8FBFF
カード：rgba(255,255,255,0.86)
枠線：#DDE7F5
メイン文字：#10234A
サブ文字：#6B7A99
Blue：#2F67FF
Pink：#F044B8
Cyan：#25C3D8
Purple：#8B5CF6
Amber：#F4B63F
```

## タイポグラフィ

* カード見出しは太めにする
* 補助情報や日付は小さめ、淡い色にする
* 数値は大きく、単位は小さく表示する
* グラフや凡例は視認性を優先し、文字サイズを小さくしすぎない

## 装飾

* メイン画面に画面名だけのヘッダーは表示しない
* ヘッダー右上の点マークは表示しない
* 発光は控えめにする
* 装飾は操作要素と重ならない
* カード内の罫線は薄くし、セクション分割に使う

## 分析画面のデータ計算

必要な派生値：

* 習慣ごとの現在連続日数
* 習慣ごとの最長連続日数
* 習慣ごとの最長連続日数達成日
* 日付ごとの実施済み習慣数
* 習慣ごとの累計実施日数または累計記録量
* 対象期間内の総実施日数または総記録量
* 対象期間内の実施ユニーク日数
* 対象期間内の1日平均実施数または実施日あたり平均記録量
* 対象期間内の実施率

## 注意点

* 全体streakはMVPでは計算しない
* 未来日は分析対象に含めない
* タイムゾーンは端末のローカル日付を基準にする
* `YYYY-MM-DD` 形式の日付キーで集計する
* グラフは横幅が狭い端末でも凡例とツールチップが重ならないようにする
